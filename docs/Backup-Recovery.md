# 💾 استراتيجية النسخ الاحتياطي واستعادة البيانات (Backup & Recovery Plan)

> **الإصدار**: 2.0 | **آخر تحديث**: 2026-06-08 | **السرية**: سرّي للغاية

تم تصميم نظام النسخ الاحتياطي في **Al-Ahram Pay** لضمان استمرارية الأعمال وحماية البيانات المالية وحسابات العملاء من الفقدان تحت أي ظرف طارئ (فشل الأجهزة، الكوارث الطبيعية، أو الأخطاء البشرية).

---

## 📊 الأهداف التشغيلية (RPO & RTO)

| المؤشر | الهدف المطلوب | آلية التحقيق |
|---|---|---|
| **RPO (حد خسارة البيانات)** | **أقل من 24 ساعة** | نسخ احتياطي يومي كامل تلقائي لقاعدة البيانات والمستندات. |
| **RTO (وقت استعادة الخدمة)** | **أقل من 30 دقيقة** | بنية تحتية جاهزة للاسترداد الفوري باستخدام حاويات Docker. |

---

## 1. النسخ الاحتياطي التلقائي (Automated Backups)

### أ. قاعدة البيانات (MongoDB Backup)
يتم استخدام أداة `mongodump` لأخذ نسخة احتياطية مضغوطة من البيانات بالكامل.

#### نص برمجي للنسخ المجدول (`backup.sh`):
```bash
#!/bin/bash
# ==============================================================================
# 📂 نص أخذ النسخة الاحتياطية لـ MongoDB (Gzip Compressed)
# ==============================================================================

# 1. إعداد المتغيرات
BACKUP_DIR="/backups"
DATE=$(date +%Y-%m-%d_%H-%M-%S)
DEST="$BACKUP_DIR/$DATE"
MONGO_URI=${MONGO_URI:-"mongodb://localhost:27017/vodafone_cash"}

echo "🕒 بدء عملية النسخ الاحتياطي: $(date)"

# 2. إنشاء مجلد النسخة
mkdir -p "$DEST"

# 3. تشغيل mongodump
mongodump --uri="$MONGO_URI" --out="$DEST" --gzip

if [ $? -eq 0 ]; then
    echo "✅ تم النسخ الاحتياطي بنجاح في مجلد: $DEST"
else
    echo "❌ فشل النسخ الاحتياطي لقاعدة البيانات!"
    exit 1
fi

# 4. تنظيف النسخ القديمة (الاحتفاظ بآخر 30 يوماً فقط)
echo "🧹 تنظيف النسخ الاحتياطية القديمة (الأقدم من 30 يوماً)..."
find /backups -maxdepth 1 -type d -mtime +30 -exec rm -rf {} +

echo "🏁 انتهت العملية بنجاح."
```

#### جدولة المهمة عبر Cron Job:
قم بإضافة السطر التالي في ملف `crontab` الخاص بالنظام للتشغيل التلقائي يومياً في تمام الساعة **2:00 صباحاً**:
```bash
0 2 * * * /bin/bash /usr/src/app/scripts/backup.sh >> /var/log/mongodb-backup.log 2>&1
```

### ب. الصور والإيصالات (Uploaded Receipts Backup)
يتم حفظ صور إثبات التحويل المرفوعة من المنفذين في المجلد `./uploads`. لضمان سلامتها، يجب نسخ هذا المجلد بشكل مستقل.

#### أمر النسخ الاحتياطي للملفات:
```bash
tar -czf /backups/uploads_$(date +%Y-%m-%d).tar.gz ./uploads
```

## 2. إجراءات استعادة البيانات (Recovery Procedures)

في حال حدوث خلل أو انهيار في النظام، يتم استرجاع البيانات وفقاً للخطوات التالية:

### الخطوة 1: تهيئة قاعدة بيانات نظيفة
تأكد من تشغيل حاوية MongoDB فارغة (سواء عبر docker-compose أو مباشرة):
```bash
docker compose -f docker-compose.prod.yml up -d mongo_db
```

### الخطوة 2: استرجاع البيانات (Restore database)
استخدم أداة `mongorestore` لاستعادة قاعدة البيانات بالكامل من النسخة المضغوطة المحددة:
```bash
# استعادة البيانات من مجلد النسخة المحددة (مثال: نسخة 2026-06-08)
mongorestore --uri="mongodb://localhost:27017/vodafone_cash" --gzip /backups/2026-06-08_02-00-00/
```

### الخطوة 3: استرجاع ملفات الإيصالات والصور
قم بفك ضغط ملف الصور الاحتياطي إلى مجلد المشروع الرئيسي:
```bash
tar -xzf /backups/uploads_2026-06-08.tar.gz -C /path/to/project/
```

### الخطوة 4: إعادة تشغيل الخدمة بالكامل
أعد تشغيل بقية مكونات النظام بعد اكتمال الاستعادة بنجاح:
```bash
docker compose -f docker-compose.prod.yml restart ahram_core_prod
```

## 3. التحقق والمطابقة (Backup Validation)

> [!WARNING]
> **النسخة الاحتياطية التي لا يتم اختبار استعادتها هي نسخة غير موجودة!**
> يجب على مسؤولي النظام إجراء اختبار استعادة دوري (مرة واحدة شهرياً على الأقل) في بيئة اختبارية معزولة للتأكد من سلامة ملفات النسخ وجاهزيتها للتشغيل.
